# 09 - Preguntas para defensa del proyecto

1. ¿Qué problema resuelve tu proyecto?

Permite organizar el proceso de QA final de un proyecto web, registrando bugs, correcciones, coevaluación, checklist y plan de mejora.

2. ¿Cuál es la funcionalidad principal?

La funcionalidad principal es registrar bugs priorizados y documentar su corrección con evidencias.

3. ¿Qué tecnologías usaste y por qué?

Usé HTML, CSS, JavaScript, Bootstrap, Vite, JSON Server, SweetAlert2 y Chart.js. Estas herramientas permiten crear una aplicación organizada, responsive, con datos guardados y visualización de métricas.

4. ¿Cómo comprobaste que JSON Server funciona?

Lo comprobé abriendo las rutas de la API en `localhost:3001` y verificando que los bugs, mejoras, coevaluaciones y checklist se guardaran correctamente.

5. ¿Qué bug fue más importante y cómo lo corregiste?

El bug más importante fue la eliminación sin confirmación. Lo corregí agregando una alerta con SweetAlert2 antes de borrar un registro.

6. ¿Qué evidencia demuestra que aplicaste correcciones?

Las capturas de antes/después, el código modificado y los documentos de informe de correcciones.

7. ¿Qué retroalimentación recibiste?

Como trabajé individualmente, realicé una autoevaluación técnica. Detecté que debía mejorar validaciones, confirmaciones y documentación.

8. ¿Qué mejora futura implementarías primero?

Implementaría autenticación para controlar quién puede modificar o eliminar registros.

9. ¿Cómo mejoraste la usabilidad o la claridad del proyecto?

Agregué confirmación antes de eliminar y completé la documentación para que el proceso sea más claro.

10. ¿Qué aprendiste del proceso de QA y mejora continua?

Aprendí que QA no es solo encontrar errores, sino documentarlos, priorizarlos, corregirlos y demostrar con evidencia que el proyecto mejoró.
