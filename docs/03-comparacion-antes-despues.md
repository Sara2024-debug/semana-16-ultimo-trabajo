# 03 - Comparación antes y después

| Corrección | Antes | Después | Archivo modificado | ¿Por qué mejora el proyecto? |
|---|---|---|---|---|
| Validación de evidencia | `evidencias/antes/evidencia-no-validada.png` | `evidencias/despues/evidencia-validada.png` | `src/js/validaciones.js` | Obliga a registrar evidencias reales y organizadas dentro de las carpetas del proyecto. |
| Confirmación antes de eliminar | `evidencias/antes/eliminar-sin-confirmacion.png` | `evidencias/despues/eliminar-confirmacion.png` | `src/js/qa.js` | Evita borrar registros por accidente y mejora la experiencia del usuario. |

## Justificación

Antes el proyecto tenía partes funcionales, pero faltaban controles importantes para calidad final. Después de los ajustes, el sistema valida mejor la evidencia, protege la eliminación de bugs y deja documentación lista para sustentar la exposición.
