# 04 - Plan de mejora final

| Mejora propuesta | Problema o necesidad que atiende | Prioridad | ¿Cómo se implementaría? | Beneficio esperado |
|---|---|---|---|---|
| Agregar autenticación | Evita que cualquier usuario modifique o elimine registros. | Alta | Crear login con usuario y contraseña. | Mayor seguridad. |
| Migrar a base de datos real | JSON Server solo sirve para pruebas locales. | Alta | Usar Firebase, Supabase o backend propio. | Proyecto más profesional. |
| Exportar reporte QA | Facilita entregar evidencias y resultados. | Media | Generar PDF o Excel con bugs, checklist y mejoras. | Mejor presentación. |
| Agregar filtros de búsqueda | Facilita encontrar bugs o mejoras específicas. | Media | Crear buscador por área, prioridad o estado. | Mejor usabilidad. |
| Mejorar accesibilidad | Permite que más usuarios puedan usar el sistema. | Media | Revisar contraste, labels, teclado y textos alternativos. | Experiencia más inclusiva. |

## Mejora más importante

La primera mejora que implementaría sería la autenticación, porque el sistema maneja registros de QA y acciones como eliminar bugs. Si cualquier persona puede entrar y modificar información, se puede perder control del proceso.

## Reflexión

El plan de mejora final demuestra que el proyecto puede seguir creciendo. Aunque funcione, todavía puede ser más seguro, más fácil de usar y más útil para una presentación profesional.
